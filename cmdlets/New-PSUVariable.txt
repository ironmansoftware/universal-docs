
NAME
    New-PSUVariable
    
SYNOPSIS
    Creates a new variable in PSU.
    
    
SYNTAX
    New-PSUVariable [-AppToken <String>] [-ComputerName <String>] [-Description <String>] [-DisableRunAsSupport] [-InputObject <Object>] [-Integrated] -Name <String> [-Type <String>] [-UseDefaultCredentials] [-Value <Object>] -Vault <String> [-Database] [-Force] [-Role <String[]>] [<CommonParameters>]

    New-PSUVariable [-AppToken <String>] [-ComputerName <String>] [-Description <String>] [-DisableRunAsSupport] [-Integrated] -Name <String> [-Type <String>] [-UseDefaultCredentials] [-Value <Object>] [-Database] [-Force] [-Role <String[]>] [<CommonParameters>]
    
    
DESCRIPTION
    Creates a new variable in PSU. Variables are available in all scripts that run within PSU.

NOTES
    
    
        
    
    -------------------------- Example 1 --------------------------
    
    PS C:\> New-UAVariable -Name 'UserName' -Value 'Adam'
    
    Creates a new variable in PSU named `UserName`. You can use the $UserName variable in all your scripts.
    -------------------------- Example 2 --------------------------
    
    PS C:\> New-UAVariable -Name 'UserName' -Value 'Adam' -Vault "Vault"
    
    Creates a new variable in the vault named `Vault`. The variable's value is not stored in PSU and will only be retrieved when running scripts.
